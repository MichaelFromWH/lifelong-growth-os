# Skill: Current Stage Assessment

Use this skill when the goal exists but the user's current position is uncertain.

## Purpose

Locate where the user is now with enough basis to generate a useful route.

## Core Rule

The universal diagnostic object is current position, not always current level. Current level is only one useful lens for skill training, games, exams, and other measurable performance domains.

## When To Use

- A goal contract exists but current stage is unknown.
- The user self-rates or gives a vague description without useful basis.
- The plan depends on knowing constraints, lifecycle stage, weak points, available assets, decision context, or baseline performance.
- A route would be irresponsible without knowing where the user currently stands.

## Assessment Modes

Choose the lightest mode that provides enough signal.

| Mode | Best For | Example |
| --- | --- | --- |
| Diagnostic Q&A | Fuzzy, strategic, career, or decision goals. | Ask about context, constraints, options, and unresolved questions. |
| Artifact review | Writing, design, product, creative, or project goals. | Review a past work sample, prototype, plan, or asset list. |
| Mini task | Skills, execution, craft, and project goals. | Ask the user to write, solve, design, speak, decide, or build a small sample. |
| Score or benchmark review | Games, exams, fitness, sales, business, or measurable performance. | Interpret a rank, score report, KPI, replay, or baseline metric. |
| Routine audit | Habits and life-practice goals. | Review environment, triggers, routines, friction, and support systems. |
| Context interview | Identity growth, motivation, or life direction. | Explore values, tensions, constraints, and repeated patterns. |

## Current-Position Lenses

- Skill-training goals: level, benchmarks, practice history, weak points, feedback quality.
- Project goals: lifecycle stage, validated assumptions, available assets, blockers, next uncertainty.
- Strategy or life goals: constraints, values, decision context, risks, unresolved questions.
- Habit goals: environment, triggers, routine structure, friction, support systems.
- Research goals: knowledge map, core questions, information gaps, judgment criteria.

## Process

1. Select assessment mode based on goal category.
2. Explain what basis you need and why.
3. Ask for existing evidence first; do not create a test if the user already has useful artifacts.
4. Run a short assessment: usually 3 to 7 questions or one small task.
5. Identify strengths, weaknesses, prerequisites, blockers, and confidence.
6. If confidence is low, mark missing basis rather than overclaiming.

## Output Format

```markdown
## Current Stage

- Stage:
- Confidence:
- Assessment basis:
- Strengths:
- Weaknesses:
- Missing prerequisites:
- Distance to target:
- Recommended next assessment, if needed:
```

## Quality Bar

A good assessment is:

- Basis-backed.
- Proportionate to the goal.
- Clear about confidence.
- Useful for planning the next step.
- Not humiliating, overly clinical, or falsely precise.
