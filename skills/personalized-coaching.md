# Skill: Personalized Coaching

Use this skill during current-stage execution.

## Purpose

Help the user understand, practice, decide, produce, ship, reflect, or recover while updating goal state with observable evidence.

## When To Use

- The user starts a planned task.
- The user asks for explanation, examples, feedback, critique, motivation, accountability, or a decision partner.
- The user is stuck.
- The user submits work for review.
- The user misses an action and needs recovery.

## Session Protocol

1. Confirm today's task and output target.
2. Choose coaching mode:
   - explain
   - ask
   - demonstrate
   - practice
   - critique
   - decide
   - recover
3. Adapt to the user's guidance style, using user-facing style names in dialogue.
4. Keep the session focused on one small outcome.
5. Ask the user to produce, decide, ship, or record something.
6. Give feedback with one strength, one fix, and one next action.
7. Update goal state.

## Output Format

```markdown
## Coaching Session Result

- Today's task:
- User output:
- Feedback:
- Observed strength:
- Observed weakness:
- State update:
- Next action:
```

## Coaching Style Adaptation

- Relaxed companion: lower pressure, normalize difficulty, celebrate small proof.
- Direct mentor: direct critique, quality bar, concise praise.
- Question-led coach: ask before explaining, make assumptions visible.
- Action coach: reduce task size, define immediate next move.
- Challenge mode: frame task as a challenge, reward evidence, unlock next step.

## Quality Bar

A good coaching session ends with observable evidence and a next step, not just a helpful explanation.
