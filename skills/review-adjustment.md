# Skill: Review Adjustment

Use this skill at review points or when progress evidence suggests the route or plan should change.

## Purpose

Compare evidence against the goal contract, update the goal state, and decide whether to continue, adjust, upgrade, pause, complete, or define the next version.

## When To Use

- A scheduled review date arrives.
- The user completes a stage.
- The user repeatedly misses actions.
- The user progresses faster than expected.
- New evidence contradicts the old assessment.
- The user's situation, constraints, or goal changes.
- The user asks whether they have achieved the goal.

## Process

1. Gather evidence since the last review.
2. Compare it with the current plan and goal contract.
3. Identify what changed: capability, motivation, constraints, preference, risk, or goal.
4. Update current stage and confidence.
5. Decide:
   - continue current plan
   - adjust workload
   - change route
   - change guidance style
   - upgrade target
   - pause
   - complete
   - define next version
6. Produce the next plan or next review rule.

## Output Format

```markdown
## Review Result

- Evidence reviewed:
- Progress judgment:
- Updated current stage:
- Confidence:
- What worked:
- What did not work:
- Decision:
- Plan adjustment:
- Next review date:
```

## Quality Bar

A good review is honest without being discouraging. It should change the plan when evidence demands it and preserve momentum when the user is on track.
