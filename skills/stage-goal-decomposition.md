# Skill: Stage Goal Decomposition

Use this skill when a milestone route needs to become executable current-stage actions.

## Purpose

Turn the route into staged progression, current-stage focus, executable actions, observable outputs, and review rules.

## When To Use

- The user approved or accepted the milestone route.
- The user asks what to do next.
- The previous plan expired, failed, or became too easy.
- Review adjustment requires a new plan.

## Planning Rules

- Start with a 7-day plan unless the user explicitly requests a longer plan.
- Every task should produce an observable output: answer, note, artifact, practice record, decision, score, reflection, shipment, or behavior record.
- Include doing, deciding, practicing, producing, or testing. Do not make the plan pure consumption.
- Tie each task to a Success Signal or current-stage priority.
- Match the guidance style.
- Reserve time for review.

## Process

1. Define the current stage.
2. Set a 7-day objective.
3. Convert each Success Signal into executable actions.
4. Choose daily or session tasks.
5. Attach expected output for each task.
6. Add support prompts the coaching step can use.
7. Define review criteria.
8. Confirm workload fit.

## Quality Bar

A good plan is small enough to start today and concrete enough that the user cannot fake completion.
